01 Wireshark ▼
شنو هو؟ Wireshark هو network packet analyzer — يلتقط ويحلل كل الـpackets اللي تعدي على الشبكة في real-time. نقاط مهمة
  • يمكن استعماله لفحص headers وpayloads، وفهم الطبقات (L2-L7).
  • عند تحليل ترافيك مشفر (HTTPS/SSH) تحتاج مفاتيح أو فك تشفير TLS لرؤية المحتوى.
  • احذر من حفظ أو مشاركة الـcaptures لأنه يحتوي على بيانات حساسة (passwords، tokens).
كيفاش يشتغل؟ يحط الـnetwork interface في promiscuous mode ← يلتقط كل الـframes ← يعرض محتوياتها (protocol, source, destination, data). الفرق: SSH مقابل Telnet
  • SSH — كل البيانات مشفرة، Wireshark يشوف الـpackets بصح ما يقرا محتواهم
  • Telnet — بيانات plain text، Wireshark يشوف حتى الـpassword مباشرة!
استعمالاته تشخيص مشاكل الشبكة، كشف الهجمات، تعلم البروتوكولات. ❓ Wireshark use? Captures and analyzes network packets in real-time — يستخدم لـ network troubleshooting، protocol analysis، وكشف الهجمات. ❓ What can be seen when capturing Telnet vs SSH? مع Telnet، كل البيانات (including passwords) visible بـplain text في الـcapture. مع SSH، كل شي مشفر والـattacker يشوف فقط الـpackets بدون فهم محتواهم.
02 Nmap ▼
شنو هو؟ Network Mapper — أداة مفتوحة المصدر لـport scanning واكتشاف الأجهزة على الشبكة. كيفاش يشتغل؟ يبعث packets خاصة للـhost ← يحلل الردود ← يعرف: شكون على الشبكة، شنو الـports المفتوحة، شنو الـservices الشغالة، شنو الـOS. أهم الـOptions
  • -sS SYN Scan (stealth) — ما يكملش الـhandshake
  • -O OS detection
  • -sV Service version detection
  • -p تحديد ports معينة
الاستعمال Penetration testing، network auditing، security assessment. نصائح عملية
  • ابدأ بـping sweep ثم استخدم -sV و -O للحصول على صورة أوضح عن الخدمات والـOS.
  • استعمل خيارات الإخفاء وعدّل الـtiming (`-T`) لتقليل الاكتشاف من أنظمة الـIDS.
  • حترم القوانين — لا تقم بسكان شبكات بدون إذن صريح.
❓ Nmap purpose? Network discovery and port scanning — يحدد الـhosts المتصلة، الـports المفتوحة، الـservices والـOS على الشبكة. ❓ الفرق بين -sS و -sT في Nmap؟ -sS = SYN scan (stealth) — ما يكملش TCP handshake، أسرع وأقل noisy. -sT = full connect scan — يكمل الـhandshake بالكامل.
03 Shodan ▼
شنو هو؟ موتور بحث للأجهزة المتصلة بالإنترنت — servers، cameras، routers، IoT devices. مش زي Google اللي يبحث في المواقع. كيفاش يشتغل؟ يسكان الإنترنت باستمرار ← يجمع بيانات الـbanners (المعلومات اللي ترجع من الخدمات) ← يفهرسها في قاعدة بيانات قابلة للبحث. شنو تلقى فيه؟
  • Devices مفتوحة بدون password
  • Servers ضعيفة (vulnerable services)
  • Cameras صناعية أو منزلية مكشوفة
كيفاش تحمي راسك منه؟ تغلق الـports غير الضرورية، تستعمل firewall، تحمي الخدمات بـpassword وتشفير. ملاحظات
  • Shodan يظهر بيانات الـbanners — تحديث الخدمات وإخفاء الـbanners يقلل من المخاطر.
  • افحص دورياً نطاقك باستخدام Shodan وتحلّ بالاستجابة السريعة لإصلاح المشكلات.
❓ Shodan main function? Search engine for internet-connected devices — يفهرس servers، cameras، routers، IoT devices المكشوفة بدون حماية. ❓ كيفاش Shodan تجمع البيانات؟ Shodan تسكان الإنترنت باستمرار، تجمع الـbanners (معلومات من الخدمات)، وتفهرسها في قاعدة بيانات قابلة للبحث.
04 ARP Protocol & Simulation ▼
شنو هو ARP؟ Address Resolution Protocol — يحول الـIP address إلى MAC address داخل نفس الشبكة (LAN). كيفاش يشتغل؟
  • الجهاز A عنده IP ديال B، يبغي MAC ديالها
  • يبعث ARP Request broadcast: "شكون عنده هاذ الـIP؟"
  • الجهاز B يرد بـARP Reply: "أنا، وهاذ هو MAC ديالي"
  • الجهاز A يخزن الجواب في الـARP Cache
Packet Tracer نشبه هاذ العملية بـsimulation — نشوف الـpackets وهي تمشي بين الأجهزة خطوة خطوة. نقاط عملية
  • الـARP مفيد للتعلم لكنه عرضة للتزوير؛ راقب التغييرات المفاجئة في الـARP cache.
  • تأكد من فهم broadcast domain والـswitch behavior عند محاكاة الشبكات.
❓ ARP protocol شنو الهدف ديالو؟ تحويل IP addresses إلى MAC addresses في نفس الشبكة (LAN) — ضروري حتى يقدر الـdevices توصل بعضهم.
05 ARP Poisoning ▼
شنو هو؟ هجوم يبعث فيه المهاجم ARP Replies مزورة على الشبكة باش يربط MAC ديالو بـIP ديال شخص ثاني. النتيجة؟ الأجهزة ترسل بياناتها للمهاجم بدل الوجهة الحقيقية ← Man-in-the-Middle (MitM) مثال
  • المهاجم يقول للـPC: "أنا هو الـGateway"
  • يقول للـGateway: "أنا هو الـPC"
  • كل الترافيك يعدي عليه!
الحماية Dynamic ARP Inspection (DAI)، Static ARP entries، HTTPS. ممارسات وقائية
  • فعّل DAI على الـswitches المدعومة، واستعمل static ARP للمواد الحساسة فقط.
  • راقب سجلات الشبكة لاكتشاف تحديثات ARP غير متوقعة أو تكرار عناوين MAC.
❓ ARP poisoning شنو النتيجة ديالو على الشبكة؟ Man-in-the-Middle (MitM) attack — كل الترافيك بين الضحية والـgateway يعدي عند المهاجم يقدر يسرقو ويعدلو.
06 Brute Force Attack ▼
شنو هو؟ هجوم يجرب فيه المهاجم كل الاحتمالات الممكنة للـpassword أو الـkey حتى يلقى الصح. الأنواع
  • Pure Brute Force — يجرب كل التركيبات (abc, abd, abe...)
  • Dictionary Attack — يستعمل قائمة passwords شائعة
  • Rainbow Table — يستعمل hashes محسوبة مسبقاً
الحماية Account lockout، CAPTCHA، rate limiting، passwords طويلة ومعقدة، MFA. توصيات
  • استخدم مخازن كلمات مرور قوية للـhashing مع salt (bcrypt/argon2) لتقليل فعالية هجمات الـrainbow.
  • راقب نمط المحاولات واحذر من المحاولات الموزّعة عبر عدة نقاط (distributed brute-force).
❓ Characteristic of brute-force? يـexhaustive وguaranteed يلقى الـpassword في الآخر، لكن slow جداً مع passwords طويلة ومعقدة. ❓ Dictionary attack vs Pure Brute Force؟ Dictionary = استخدام قائمة passwords شائعة (أسرع لكن قد يفشل). Pure Brute = كل التركيبات الممكنة (أبطأ لكن guaranteed يشتغل).
07 Evil Twin Attack ▼
شنو هو؟ المهاجم يدير نقطة WiFi مزورة بنفس اسم الشبكة الحقيقية (SSID) وسيغنال أقوى — الضحية تتوصل بيها بدون ما تعرف. كيفاش يشتغل؟
  • المهاجم يسكان الشبكات الموجودة
  • يدير AP مزور بنفس الـSSID
  • يقوي الـsignal → الضحية تتوصل به أوتوماتيك
  • يسرق البيانات أو يعمل MitM
الهدف سرقة credentials، intercept traffic، تركيب malware. الحماية تحقق من الـcertificate (HTTPS)، استعمل VPN، تجنب الـWiFi العام. ملاحظات
  • تفعيل 802.1X وWPA3 حيثما أمكن يقلل من خطر Evil Twin.
  • لا تقم بالاتصال التلقائي بشبكات محلية محفوظة دون التأكد من الـBSSID.
❓ Evil Twin attack كيفاش الضحية توقع فيه؟ الضحية جهازها يفتش عن الشبكة اللي كانت متوصلة فيها قبل (remembered networks). المهاجم يدير AP بنفس الـSSID والـsignal أقوى فالضحية توصل به بدون ما تسأل.
08 SQL Injection ▼
شنو هو؟ المهاجم يحقن كود SQL خبيث في مدخلات التطبيق ← يغير منطق الـquery ← يتحكم في قاعدة البيانات. مثال 
username: admin' OR '1'='1
-- Result: query always returns TRUE → login without password!
الأنواع
  • In-band — يرجع النتائج مباشرة
  • Blind — ما يرجع شي بصح يقدر يستنتج
  • Out-of-band — يستعمل قناة ثانية
الحماية Prepared statements، Parameterized queries، Input validation، Least privilege. ممارسات إضافية
  • سجل وروديات الـqueries المشبوهة وحطّ حد للمدخلات غير المتوقعة.
  • اعمل فحص دوري للـORM والـframework المستخدم لأن بعض الإعدادات قد تعطل الحماية التلقائية.
❓ SQL Injection types - In-band vs Blind vs Out-of-band؟ In-band = النتائج visible مباشرة. Blind = ما يرجع نتائج بصح المهاجم يقدر يستنتج بـtrue/false tests. Out-of-band = يستعمل قناة ثانية (مثلاً DNS exfiltration).
09 XSS – Cross-Site Scripting ▼
شنو هو؟ المهاجم يحقن JavaScript خبيث في صفحة ويب ← يتنفذ في browser ديال الضحية. الأنواع
  • Stored XSS — الـscript يتخزن في الـDB ويتعرض لكل الزوار
  • Reflected XSS — الـscript في الـURL يرجع مباشرة
  • DOM-based — يغير الـDOM مباشرة بدون سيرفر
الخطر سرقة cookies/sessions، keylogging، redirect لمواقع خبيثة. الحماية Input sanitization، Output encoding، Content Security Policy (CSP)، HttpOnly cookies. نصائح عملية
  • استعمل CSP صارم يمنع تحميل السكربتات من مصادر غير موثوقة ويقلل من أثر XSS.
  • اجعل كل الكوكيز الحساسة `Secure` و`HttpOnly` وقيّم استخدام SameSite.
❓ XSS types - Stored vs Reflected vs DOM-based؟ Stored = الـscript يتخزن في الـDB وينفذ لكل المستخدمين. Reflected = في الـURL ويرجع مباشرة في الـHTML. DOM-based = يغير الـDOM على الـclient side بدون server involvement.
10 Malware – Virus, Trojan, Ransomware ▼
الفروقات الأساسية
  • Virus — يرفق بـfiles ويتكاثر، يحتاج تنفيذ بشري
  • Worm — يتكاثر وحده بدون مساعدة
  • Trojan — يتنكر كـsoftware شرعي (الحصان الطروادة)
  • Ransomware — يشفر الملفات ويطلب فدية
  • Spyware — يتجسس ويسرق معلومات
  • Rootkit — يخفي نفسه ويتحكم في النظام
❓ Malware categories - أي واحد يتكاثر تلقائي؟ Worm — يتكاثر وحده بدون تدخل بشري (عكس Virus اللي يحتاج تنفيذ يدوي). Virus و Worm كلهم malware بصح Worm أخطر. الموضوع: إنشاء فيروس وإضافته لتطبيق الـTrojan نموذج — يبدو تطبيق عادي لكنه يحتوي كود خبيث في الخلفية. ملاحظات أمنية
  • لا تختبر تطوير أو نشر برمجيات خبيثة على أنظمة حية — استخدم بيئات معزولة (VMs/labs).
  • تعلّم كيف تتعرف الأدوات الأمنية على الـmalware (signatures, heuristics) لتحسين الدفاع.
11 RBAC – Role-Based Access Control ▼
شنو هو؟ نظام صلاحيات يعطي كل مستخدم حقوق بناءً على دوره (Role) وليس هويته الشخصية. المبادئ
  • Least Privilege — كل شخص ياخذ الحد الأدنى من الصلاحيات اللي يحتاجها
  • Role = مجموعة صلاحيات: Admin, User, Moderator
  • Separation of Duties — تفريق المهام الحساسة
مثال Admin ← كل الصلاحيات | Editor ← قراءة وكتابة | Viewer ← قراءة فقط الفائدة سهولة الإدارة، تقليل الخطر، امتثال أسهل للـcompliance. نقاط تنفيذية
  • راجع الأذونات دورياً وتحقّق من عدم تزايد صلاحيات الأدوار بمرور الوقت (privilege creep).
  • استخدم سياسات مبنية على الدور مع قواعد واضحة لإدارة كيفية إضافة/تعديل الأدوار.
❓ RBAC principle - Least Privilege شنو معناه؟ كل مستخدم يأخذ الحد الأدنى من الـpermissions اللي يحتاجها لـrole ديالو. مثلاً Viewer ما يقدر يعدل، Admin بس يقدر يحذف.
12 Metasploit ▼
شنو هو؟ Framework متكامل للـpenetration testing — يحتوي آلاف الـexploits والـpayloads جاهزة للاستخدام. المفاهيم الأساسية
  • Exploit — الكود اللي يستغل الثغرة
  • Payload — ما يتنفذ بعد نجاح الاستغلال (مثلاً reverse shell)
  • Module — وحدة قابلة للاستخدام (auxiliary, exploit, post)
  • msfconsole — واجهة الأوامر
الفرق: Exploit مقابل Payload Exploit = الباب اللي تدخل منه | Payload = شنو دخلت تدير الاستعمال الشرعي Penetration testing مرخص، اختبار أنظمة الحماية. ممارسات آمنة
  • اجعل استخدام Metasploit داخل بيئة اختبار مع موافقة كتابية، وتوثيق كل الاختبارات والنتائج.
  • احذر من تشغيل payloads على أنظمة إنتاجية أو مشاركة شفرات خطرة دون رقابة.
❓ Exploit vs Payload في Metasploit؟ Exploit = الكود اللي يستغل الثغرة (الباب). Payload = ما يتنفذ بعد النجاح (reverse shell, Meterpreter, etc).
13 IDS / IPS – Suricata & Zeek ▼
IDS مقابل IPS
  • IDS Intrusion Detection System — يكشف الهجمات ويرسل تنبيه فقط (passive)
  • IPS Intrusion Prevention System — يكشف ويبلوك الهجمات (active)
Suricata أداة مفتوحة المصدر تشتغل كـIDS وIPS وNSM. تحلل الترافيك بـrules وتكشف الهجمات في real-time. يدعم وضعين: IDS (يسجل فقط) وIPS (يبلوك). Zeek (Bro سابقاً) Network Security Monitor — مش IDS تقليدي، يحلل الترافيك ويولد logs مفصلة (connections, DNS, HTTP). Focus على الـanalysis والـbehavior مش الـsignatu ❓ IPS advantage over IDS؟ IPS actively blocks/drops الـmalicious traffic في real-time. IDS فقط detects ويرسل alerts (passive). IPS = IDS + blocking.res. الفرق: Suricata مقابل Zeek Suricata = detection/blocking بـsignatures | Zeek = تحليل عميق للـbehavior نقاط تشغيلية
  • اجمع logs من كليهما للحصول على صورة متكاملة: Suricata للـalerts وZeek للـcontext والتحليل.
  • حدّث قواعد الـIDS/IPS بانتظام وقلّل الـfalse positives عبر تهيئة القواعد حسب البيئة.
14 SQL Injection – Simulation & Prevention ▼
Simulation استعمال أدوات مثل sqlmap أو Burp Suite لاختبار الثغرات في بيئة lab آمنة. أفضل ممارسات الحماية
  • Prepared Statements — تفصل الكود عن البيانات
  • Input Validation — ترفض المدخلات الخطرة
  • WAF — Web Application Firewall
  • Least Privilege — الـDB user ماعندوش أكثر من اللي يحتاج
  • تجنب Dynamic SQL — ما تبنيش queries من strings مباشرة
❓ Prevent SQL injection؟ Use Prepared Statements / Parameterized Queries — تفصل الـSQL code عن الـuser data بحيث الـinjection مستحيلة. تأكيد وفحص
  • ادمج فحوصات الأمان ضمن CI/CD (SAST/DAST) لاكتشاف نقاط الضعف مبكراً.
  • نفّذ اختبارات دورية باستخدام أدوات أوتوماتيكية مثل sqlmap في بيئة مختبرة.
15 OpenSSL – Chiffrement & Hachage ▼
شنو هو OpenSSL؟ مكتبة مفتوحة المصدر توفر أدوات للـcryptography: تشفير، hashing، SSL/TLS certificates. الـChiffrement (التشفير)
  • Symmetric — نفس المفتاح للتشفير والفك (AES)
  • Asymmetric — مفتاح عام + خاص (RSA)
الـHachage دالة تحول البيانات لـhash ثابت الطول — أحادي الاتجاه (MD5, SHA-256). الاستعمال: التحقق من سلامة الملفات، تخزين passwords. أوامر أساسية 
openssl enc -aes-256-cbc -in file.txt -out file.enc
openssl dgst -sha256 file.txt
نقاط حماية
  • لا تستخدم MD5 أو SHA1 لتخزين كلمات المرور — استعمل bcrypt أو Argon2.
  • حفظ المفاتيح الخاصة يجب أن يكون في مخزن آمن (HSM أو key vault) وليس في ملفات plain-text.
❓ OpenSSL usage? Encrypting/decrypting data، generating SSL/TLS certificates، hashing files، creating key pairs (RSA/AES operations).
16 Hardening Linux Server ▼
شنو هو؟ تقليص سطح الهجوم بتأمين الـserver — إزالة كل ما هو غير ضروري وتقوية الإعدادات. أهم الخطوات
  • تحديث النظام دائماً — apt upgrade
  • تعطيل الـservices غير المستخدمة
  • إعداد firewall (iptables / ufw)
  • تعطيل SSH root login
  • استخدام SSH keys بدل passwords
  • تفعيل fail2ban للحماية من brute force
  • إعداد SELinux أو AppArmor
❓ Most important SSH hardening practice؟ Disable SSH root login وتستخدم SSH keys بدل passwords. يقلل جداً من brute force attacks. إجراءات إضافية
  • استعمل auditd وسجلات مركزية (syslog/ELK) لمراقبة محاولات الدخول وفشلها.
  • قيّم أمان الخدمات بعد كل تحديث أو تركيب برمجيات جديدة.
17 Nessus – Vulnerability Scanner ▼
شنو هو؟ Vulnerability Assessment Tool — يسكان الأنظمة والشبكات ويكشف الثغرات الأمنية المعروفة. كيفاش يشتغل؟
  • يسكان الـports والـservices
  • يقارن مع قاعدة بيانات CVEs
  • يعطي تقرير مفصل مع severity (Critical / High / Medium / Low)
الفرق مع Nmap Nmap = اكتشاف فقط | Nessus = اكتشاف + تقييم الثغرات + توصيات ❓ Nessus vs Nmap - الفرق؟ Nmap = network discovery و port scanning فقط (detection). Nessus = vulnerability assessment (detection + severity ranking + remediation recommendations). الاستعمال Vulnerability assessment، compliance checks، penetration testing. إستراتيجية الفحص
  • ابدأ بعمل baseline للتعرّف على الثغرات الحرجة ثم رتّب الأولويات حسب الـrisk وimpact.
  • ادمج نتائج Nessus مع نظام تتبع الثغرات لمتابعة الإصلاح والتحقق من الإغلاق.
