01 Wireshark ▼
شنو هو؟ Wireshark هو network packet analyzer — يلتقط ويحلل كل الـpackets اللي تعدي على الشبكة في real-time. كيفاش يشتغل؟ يحط الـnetwork interface في promiscuous mode ← يلتقط كل الـframes ← يعرض محتوياتها (protocol, source, destination, data). الفرق: SSH مقابل Telnet
  • SSH — كل البيانات مشفرة، Wireshark يشوف الـpackets بصح ما يقرا محتواهم
  • Telnet — بيانات plain text، Wireshark يشوف حتى الـpassword مباشرة!
استعمالاته تشخيص مشاكل الشبكة، كشف الهجمات، تعلم البروتوكولات.
02 Nmap ▼
شنو هو؟ Network Mapper — أداة مفتوحة المصدر لـport scanning واكتشاف الأجهزة على الشبكة. كيفاش يشتغل؟ يبعث packets خاصة للـhost ← يحلل الردود ← يعرف: شكون على الشبكة، شنو الـports المفتوحة، شنو الـservices الشغالة، شنو الـOS. أهم الـOptions
  • -sS SYN Scan (stealth) — ما يكملش الـhandshake
  • -O OS detection
  • -sV Service version detection
  • -p تحديد ports معينة
الاستعمال Penetration testing، network auditing، security assessment.
03 Shodan ▼
شنو هو؟ موتور بحث للأجهزة المتصلة بالإنترنت — servers، cameras، routers، IoT devices. مش زي Google اللي يبحث في المواقع. كيفاش يشتغل؟ يسكان الإنترنت باستمرار ← يجمع بيانات الـbanners (المعلومات اللي ترجع من الخدمات) ← يفهرسها في قاعدة بيانات قابلة للبحث. شنو تلقى فيه؟
  • Devices مفتوحة بدون password
  • Servers ضعيفة (vulnerable services)
  • Cameras صناعية أو منزلية مكشوفة
كيفاش تحمي راسك منه؟ تغلق الـports غير الضرورية، تستعمل firewall، تحمي الخدمات بـpassword وتشفير.
04 ARP Protocol & Simulation ▼
شنو هو ARP؟ Address Resolution Protocol — يحول الـIP address إلى MAC address داخل نفس الشبكة (LAN). كيفاش يشتغل؟
  • الجهاز A عنده IP ديال B، يبغي MAC ديالها
  • يبعث ARP Request broadcast: "شكون عنده هاذ الـIP؟"
  • الجهاز B يرد بـARP Reply: "أنا، وهاذ هو MAC ديالي"
  • الجهاز A يخزن الجواب في الـARP Cache
Packet Tracer نشبه هاذ العملية بـsimulation — نشوف الـpackets وهي تمشي بين الأجهزة خطوة خطوة.
05 ARP Poisoning ▼
شنو هو؟ هجوم يبعث فيه المهاجم ARP Replies مزورة على الشبكة باش يربط MAC ديالو بـIP ديال شخص ثاني. النتيجة؟ الأجهزة ترسل بياناتها للمهاجم بدل الوجهة الحقيقية ← Man-in-the-Middle (MitM) مثال
  • المهاجم يقول للـPC: "أنا هو الـGateway"
  • يقول للـGateway: "أنا هو الـPC"
  • كل الترافيك يعدي عليه!
الحماية Dynamic ARP Inspection (DAI)، Static ARP entries، HTTPS.
06 Brute Force Attack ▼
شنو هو؟ هجوم يجرب فيه المهاجم كل الاحتمالات الممكنة للـpassword أو الـkey حتى يلقى الصح. الأنواع
  • Pure Brute Force — يجرب كل التركيبات (abc, abd, abe...)
  • Dictionary Attack — يستعمل قائمة passwords شائعة
  • Rainbow Table — يستعمل hashes محسوبة مسبقاً
الحماية Account lockout، CAPTCHA، rate limiting، passwords طويلة ومعقدة، MFA.
07 Evil Twin Attack ▼
شنو هو؟ المهاجم يدير نقطة WiFi مزورة بنفس اسم الشبكة الحقيقية (SSID) وسيغنال أقوى — الضحية تتوصل بيها بدون ما تعرف. كيفاش يشتغل؟
  • المهاجم يسكان الشبكات الموجودة
  • يدير AP مزور بنفس الـSSID
  • يقوي الـsignal → الضحية تتوصل به أوتوماتيك
  • يسرق البيانات أو يعمل MitM
الهدف سرقة credentials، intercept traffic، تركيب malware. الحماية تحقق من الـcertificate (HTTPS)، استعمل VPN، تجنب الـWiFi العام.
08 SQL Injection ▼
شنو هو؟ المهاجم يحقن كود SQL خبيث في مدخلات التطبيق ← يغير منطق الـquery ← يتحكم في قاعدة البيانات. مثال 
username: admin' OR '1'='1
-- Result: query always returns TRUE → login without password!
الأنواع
  • In-band — يرجع النتائج مباشرة
  • Blind — ما يرجع شي بصح يقدر يستنتج
  • Out-of-band — يستعمل قناة ثانية
الحماية Prepared statements، Parameterized queries، Input validation، Least privilege.
09 XSS – Cross-Site Scripting ▼
شنو هو؟ المهاجم يحقن JavaScript خبيث في صفحة ويب ← يتنفذ في browser ديال الضحية. الأنواع
  • Stored XSS — الـscript يتخزن في الـDB ويتعرض لكل الزوار
  • Reflected XSS — الـscript في الـURL يرجع مباشرة
  • DOM-based — يغير الـDOM مباشرة بدون سيرفر
الخطر سرقة cookies/sessions، keylogging، redirect لمواقع خبيثة. الحماية Input sanitization، Output encoding، Content Security Policy (CSP)، HttpOnly cookies.
10 Malware – Virus, Trojan, Ransomware ▼
الفروقات الأساسية
  • Virus — يرفق بـfiles ويتكاثر، يحتاج تنفيذ بشري
  • Worm — يتكاثر وحده بدون مساعدة
  • Trojan — يتنكر كـsoftware شرعي (الحصان الطروادة)
  • Ransomware — يشفر الملفات ويطلب فدية
  • Spyware — يتجسس ويسرق معلومات
  • Rootkit — يخفي نفسه ويتحكم في النظام
الموضوع: إنشاء فيروس وإضافته لتطبيق الـTrojan نموذج — يبدو تطبيق عادي لكنه يحتوي كود خبيث في الخلفية.
11 RBAC – Role-Based Access Control ▼
شنو هو؟ نظام صلاحيات يعطي كل مستخدم حقوق بناءً على دوره (Role) وليس هويته الشخصية. المبادئ
  • Least Privilege — كل شخص ياخذ الحد الأدنى من الصلاحيات اللي يحتاجها
  • Role = مجموعة صلاحيات: Admin, User, Moderator
  • Separation of Duties — تفريق المهام الحساسة
مثال Admin ← كل الصلاحيات | Editor ← قراءة وكتابة | Viewer ← قراءة فقط الفائدة سهولة الإدارة، تقليل الخطر، امتثال أسهل للـcompliance.
12 Metasploit ▼
شنو هو؟ Framework متكامل للـpenetration testing — يحتوي آلاف الـexploits والـpayloads جاهزة للاستخدام. المفاهيم الأساسية
  • Exploit — الكود اللي يستغل الثغرة
  • Payload — ما يتنفذ بعد نجاح الاستغلال (مثلاً reverse shell)
  • Module — وحدة قابلة للاستخدام (auxiliary, exploit, post)
  • msfconsole — واجهة الأوامر
الفرق: Exploit مقابل Payload Exploit = الباب اللي تدخل منه | Payload = شنو دخلت تدير الاستعمال الشرعي Penetration testing مرخص، اختبار أنظمة الحماية.
13 IDS / IPS – Suricata & Zeek ▼
IDS مقابل IPS
  • IDS Intrusion Detection System — يكشف الهجمات ويرسل تنبيه فقط (passive)
  • IPS Intrusion Prevention System — يكشف ويبلوك الهجمات (active)
Suricata أداة مفتوحة المصدر تشتغل كـIDS وIPS وNSM. تحلل الترافيك بـrules وتكشف الهجمات في real-time. يدعم وضعين: IDS (يسجل فقط) وIPS (يبلوك). Zeek (Bro سابقاً) Network Security Monitor — مش IDS تقليدي، يحلل الترافيك ويولد logs مفصلة (connections, DNS, HTTP). Focus على الـanalysis والـbehavior مش الـsignatures. الفرق: Suricata مقابل Zeek Suricata = detection/blocking بـsignatures | Zeek = تحليل عميق للـbehavior
14 SQL Injection – Simulation & Prevention ▼
Simulation استعمال أدوات مثل sqlmap أو Burp Suite لاختبار الثغرات في بيئة lab آمنة. أفضل ممارسات الحماية
  • Prepared Statements — تفصل الكود عن البيانات
  • Input Validation — ترفض المدخلات الخطرة
  • WAF — Web Application Firewall
  • Least Privilege — الـDB user ماعندوش أكثر من اللي يحتاج
  • تجنب Dynamic SQL — ما تبنيش queries من strings مباشرة
15 OpenSSL – Chiffrement & Hachage ▼
شنو هو OpenSSL؟ مكتبة مفتوحة المصدر توفر أدوات للـcryptography: تشفير، hashing، SSL/TLS certificates. الـChiffrement (التشفير)
  • Symmetric — نفس المفتاح للتشفير والفك (AES)
  • Asymmetric — مفتاح عام + خاص (RSA)
الـHachage دالة تحول البيانات لـhash ثابت الطول — أحادي الاتجاه (MD5, SHA-256). الاستعمال: التحقق من سلامة الملفات، تخزين passwords. أوامر أساسية 
openssl enc -aes-256-cbc -in file.txt -out file.enc
openssl dgst -sha256 file.txt
16 Hardening Linux Server ▼
شنو هو؟ تقليص سطح الهجوم بتأمين الـserver — إزالة كل ما هو غير ضروري وتقوية الإعدادات. أهم الخطوات
  • تحديث النظام دائماً — apt upgrade
  • تعطيل الـservices غير المستخدمة
  • إعداد firewall (iptables / ufw)
  • تعطيل SSH root login
  • استخدام SSH keys بدل passwords
  • تفعيل fail2ban للحماية من brute force
  • إعداد SELinux أو AppArmor
17 Nessus – Vulnerability Scanner ▼
شنو هو؟ Vulnerability Assessment Tool — يسكان الأنظمة والشبكات ويكشف الثغرات الأمنية المعروفة. كيفاش يشتغل؟
  • يسكان الـports والـservices
  • يقارن مع قاعدة بيانات CVEs
  • يعطي تقرير مفصل مع severity (Critical / High / Medium / Low)
الفرق مع Nmap Nmap = اكتشاف فقط | Nessus = اكتشاف + تقييم الثغرات + توصيات الاستعمال Vulnerability assessment، compliance checks، penetration testing.
